Data Cleaning -- manipulation description

The Data Cleaning part is basically done by using two approaches.

1. To make sure a smooth data cleaning process, it is better to check and correct format errors in Excel document before using data cleaning tools like OpenRefine

Additional Blank Cells

As showed in the picture, there were some unexpected consecutive blank cells within columns, which made several records out of places. Hence, these blank cells were deleted to enable data alignment.

Such as rows from 180 to 454.

Unexpected Seperation

Also, the error of one project ID were separated into two cells occurs in different places. Then, these separated cells were merged in Excel document manually.

Such as row 652 to 654, 1901 to 1904.

2. After obtaining a more structured dataset, some further cleaning steps then could proceed with OpenRefine.

Spelling Error

By using Text Facet function in Agency Name column, it is obviously that there were some spelling errors need to be corrected. To fix it using cluster, cluster would display several groups in which each group included similar agency name. Then, chose reasonable grouping results and merged them into one correct name.

Such as agriculture and ‘agraculture’, commerce and ‘comerce’.

Multipul Representation

Like the former one, the error of multiple representation in Agency Name was found and resolved. (P.S. Adjusting Radius parameter can change error-tolerance of cluster function, which enable finding similar value in a larger scale. Method and Distance Function also provide more choice to explore cluster for different processing requirements.

Example: ‘agriculture department’, ‘department of agriculture’. Similar issue with transportation department

Inconsistent Formats

Inconsistent formats are common mistakes in datasets. In the given dataset, Project ID, Lifecycle Cost and date related columns had this issue. Then edit columns to replace some symbols and use edit cells -> common transforms to change date or number formats.(e.g.,Dates are represented in multiple ways, such as 2012-30-09 and 31/03/2012.)

Solution, applying replace command “value.replace(‘,’,’’)” in GREL and edit cells -> common transforms to change date or number formats.

Duplicates Records

When checking all records through Duplicates Facet function such as in Project ID column, some duplicate records emerged. This error was fixed by removing these repeated rows.

Such as ‘Unique Investment Identifier 007-000002213’ ‘012-000002005’ occurred twice.

Blank Rows

In the given dataset, there are some blank rows, which may make visualizing data difficult. Thus, Facet by Blank function was used to find blank records. After that, it is easy to remove these rows.(17 blank lines in total)

Inconsistent Value

In the given dataset, there are some one-to-one correspondences between columns such as Agency Name and Agency Code. This means that every agency name must match one corresponding code. But after checking Agency Name and Agency Code, there were some many-to-one records. Then changing wrong value to correct one, the error was fixed.

Transformed Data

There were two columns which were initially used to demonstrate project updated time. These then were merged into one column and transform to standard date format as other date records columns.

Solutiondescription: merged them into one column and transform to standard date format using “value +’ ’+cells[‘Updated Time’].value” in GREL.

Summation Records

Summation records (26 rows found) are not useful in data visualization part. Thus, these records were found by Facet by Blank and then removed.

Missing Records

By using Facet by Blank, some blank cells would be found in certain rows. The missing information such as some projects’ start dates should have been filled. Therefore, effort was made to find such information in order to enrich this dataset.
